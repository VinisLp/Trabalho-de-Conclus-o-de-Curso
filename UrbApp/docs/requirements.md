# Requisitos do UrbApp

## Requisitos funcionais

- RF1: Usuário pode criar conta e efetuar login.
- RF2: Usuário pode registrar um report com foto, descrição e localização.
- RF3: Reports aparecem em mapa com ícones de categoria.
- RF4: Usuário recebe pontos por cada report válido.
- RF5: Sistema de níveis baseado em pontos acumulados.
- RF6: Usuário visualiza seu histórico e pontuação.
- RF7: Administrador pode validar/excluir reports.

## Requisitos não funcionais

- RNF1: O aplicativo deve responder em menos de 2s nas ações principais.
- RNF2: Dados sensíveis devem ser transmitidos via HTTPS.
- RNF3: Interface deve ser responsiva em diferentes tamanhos de tela.
- RNF4: O backend deve suportar 1000 usuários simultâneos no MVP.
